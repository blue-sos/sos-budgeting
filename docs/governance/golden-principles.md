# Golden Principles

Owner: @platform
Last Reviewed: 2026-02-20
Status: active

## Principle Set

1. Prefer shared utilities over duplicated helpers.
2. Validate boundary data; do not rely on guessed shapes.
3. Enforce architecture boundaries mechanically.
4. Favor readable, composable abstractions.
5. Encode repeated review feedback into policy and tooling.
6. Keep plans self-contained and continuously updated.

## Expected Outcomes

- Lower architectural drift.
- Better long-run agent legibility.
- Faster iteration with fewer regressions.
